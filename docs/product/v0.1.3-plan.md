# v0.1.3 Case Loop Plan

> Goal: move from in-memory business skeleton to a manual case-service loop with persistence.

## Why this version exists

v0.1.2 proves the API shape and non-AI business loop. v0.1.3 must turn it into a real persistent service skeleton before AI integration.

## Scope

### Backend

- SQLAlchemy models for clients, cases, case_notes, resources
- Alembic baseline migration
- seed loader for C-0001 and resources
- repository layer replacing in-memory store
- resource matching using persisted resources
- referral model and status field
- service goal model and status field

### Frontend

- C-0001 profile page
- case timeline page
- resource list page
- note creation form
- resource matching panel

## Non-goals

- No real AI provider calls
- No final AI review gate yet
- No complex auth system
- No multi-tenant UI
- No report module

## Acceptance

- Data survives process restart.
- User can create a note and see it in timeline.
- User can run resource matching from selected tags.
- User can create a draft service goal manually.
- Referral cannot advance without an explicit consent field.

## Risk controls

- All seed data remains synthetic.
- Raw notes are preserved.
- Redacted clean text is stored separately.
- Resource matching remains deterministic.
