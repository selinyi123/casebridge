# v0.1.8 Review Apply Workflow Plan

## Goal

Move from AI draft review to controlled application preview while keeping formal case fields protected.

## Scope

### Backend

- redaction gateway interface
- redaction failure report shape
- prompt test fixture directory
- C-0001 golden note fixtures
- explicit apply endpoint design draft
- apply audit event names

### Frontend

- modified AI output editor
- apply preview panel
- explicit confirmation checkbox
- display source note and prompt version

## Non-goals

- No automatic apply
- No automatic risk classification
- No automatic resource link creation
- No provider change in this version

## Acceptance

- Accepted draft can be previewed for application.
- Formal field application remains a separate action.
- Apply action records an audit event.
- Redaction failures block provider use.

## Exit criteria

After v0.1.8, CaseBridge can demonstrate: note -> redaction -> draft -> human review -> apply preview -> explicit apply audit.
