# v0.1.9 Formal Assessment Target Plan

## Goal

Create a formal assessment target for reviewed AI drafts without allowing automatic AI mutation of case records.

## Scope

### Backend

- formal `case_assessments` target model or draft target table
- explicit apply endpoint design
- apply precondition: output review status is accepted or modified
- apply payload snapshot
- audit event: `ai.output.applied_to_assessment`
- reviewer role placeholder

### Frontend

- apply preview confirmation
- explicit checkbox: social worker reviewed and accepts responsibility
- formal assessment display panel
- link from applied assessment back to source note, AI output, prompt version, and provider

## Non-goals

- No automatic apply
- No automatic risk classification
- No automatic resource link creation
- No real external provider

## Acceptance

- Reviewed AI draft can be explicitly applied to an assessment target.
- Apply writes a formal assessment only after explicit action.
- Audit records source note, AI output, provider, prompt version, and reviewer.
- The system can still complete the workflow without AI.

## Exit criteria

After v0.1.9, CaseBridge can demonstrate: note -> redaction -> AI draft -> human review -> preview -> explicit apply -> formal assessment audit.
