# v0.1.5 AI Intake Gate Plan

> This version must start only after manual service goals and referrals exist.

## Goal

Add the first AI capability: visit-note intake analysis under strict privacy redaction, typed schema validation, and human review gate.

## Scope

### Backend

- `ai_tasks` model
- `ai_outputs` model
- AI intake output Pydantic schema
- mock provider first
- privacy redaction before provider call
- output validation
- review status: `pending`, `accepted`, `modified`, `rejected`
- no direct writes to formal case fields

### Frontend

- “Generate AI intake draft” button on note/timeline item
- AI draft panel
- accept / modify / reject actions
- clear disclaimer that AI output is draft only

## Non-goals

- No resource hallucination
- No automatic risk classification
- No automatic service plan
- No automatic referral
- No external real model until mock provider passes tests

## Acceptance

- AI output is stored only in `ai_outputs`.
- Invalid AI schema fails closed.
- Formal case records do not change until human review action.
- PII redaction runs before provider call.
- Mock provider can be swapped for a real provider later.
