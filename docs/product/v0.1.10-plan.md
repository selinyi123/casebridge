# v0.1.10 Assessment Schema and Outcome Tracking Plan

## Goal

Move from generic assessment JSON to a China-localized elderly social-work assessment schema and begin outcome tracking.

## Scope

### Backend

- assessment schema catalog
- elderly-service assessment dimensions
- assessment correction/version history draft
- outcome tracking model linked to service goals
- reviewer role placeholder

### Frontend

- structured assessment display sections
- assessment correction request UI
- outcome tracking panel
- goal progress indicators

## Non-goals

- No automatic risk classification
- No automatic service goal creation
- No automatic resource link creation
- No real external provider

## Acceptance

- Assessment data can be displayed by domain sections.
- Outcome entries can link to service goals.
- Assessment correction creates a new version instead of overwriting prior assessment.
- Audit preserves reviewer and source trace.

## Suggested dimensions

- living situation
- meal and nutrition support
- mobility and home safety
- family and informal support
- emotional and sleep clues
- client stated wishes
- missing information
- suggested follow-up
