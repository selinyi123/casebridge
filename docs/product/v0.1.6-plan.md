# v0.1.6 AI Intake Gate Plan

> This version may start only after unified timeline and audit events are usable.

## Goal

Introduce the first AI capability: structured intake analysis for a human-written case note.

## Scope

### Backend

- `ai_tasks` model
- `ai_outputs` model
- typed intake output schema
- mock provider first
- redaction before provider call
- schema validation after provider output
- review states: `pending`, `accepted`, `modified`, `rejected`
- audit events for AI draft creation and review action

### Frontend

- “Generate intake draft” action on a timeline note
- AI draft panel
- accept / modify / reject controls
- clear draft-only disclaimer

## Non-goals

- No automatic resource creation
- No automatic referral creation
- No automatic service goal creation
- No automatic risk classification
- No direct write from AI output to formal case fields

## Acceptance

- AI output is stored only in `ai_outputs`.
- Formal case records do not change until a human review action exists.
- Invalid schema fails closed.
- Redaction runs before provider call.
- Audit events record AI draft creation and review actions.

## Exit criteria

After this version, CaseBridge can demonstrate human-written note -> redacted model input -> structured AI draft -> human review -> audit trail.
