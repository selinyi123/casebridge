# v0.1.11 Assessment Versioning and Reviewer Role Plan

## Goal

Add controlled correction/version history and a small reviewer-role gate without building a full auth system yet.

## Scope

### Backend

- reviewer role catalog
- simple role guard helper
- assessment correction/version draft model
- audit event for assessment correction
- outcome report summary endpoint

### Frontend

- assessment correction history panel
- reviewer role display
- outcome report view

## Non-goals

- No full identity provider
- No real external model provider
- No automatic risk scoring
- No automatic outcome scoring

## Acceptance

- Assessment corrections create new versions rather than overwriting prior records.
- Reviewer role is visible in audit payloads.
- Outcome report summarizes manual outcomes without AI judgement.
- Existing AI output and assessment records remain traceable.
