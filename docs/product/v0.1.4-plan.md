# v0.1.4 Manual Referral and Goal Gate Plan

> This is the next key node after the C-0001 workspace becomes visible. It is still pre-AI. AI integration remains blocked until the manual service loop is complete.

## Goal

Complete the manual case-service loop by adding service goals and referral tracking with an explicit agreement-state gate.

## Why this version exists

v0.1.3 makes the client profile, case timeline, note creation, resource list, and deterministic matching visible. However, matching a resource is not the same as creating a referral, and a case note is not the same as a service plan.

v0.1.4 should add the missing manual workflow pieces before any AI endpoint is introduced.

## Scope

### Backend

- `GET /api/v1/cases/{case_id}/goals`
- `POST /api/v1/cases/{case_id}/goals`
- `GET /api/v1/cases/{case_id}/referrals`
- `POST /api/v1/cases/{case_id}/referrals`
- referral status transition guard
- agreement-state validation before referral moves beyond verification/contact stage

### Frontend

- service-goal creation form
- service-goal list
- referral creation from matched resource candidate
- referral list
- agreement-state indicator
- timeline-ready display format for goals/referrals

## Non-goals

- No real AI provider calls
- No AI-generated service plan
- No automatic referral creation
- No automatic eligibility judgement
- No formal risk classification

## Acceptance

- User can create a manual service goal.
- User can create a manual referral from an existing resource.
- Referral cannot move to `referred`, `success`, or `completed` unless agreement state is explicit.
- Matching results remain candidates only.
- Timeline can later merge notes, goals, and referrals.

## Risk controls

- Do not imply that resource matching equals referral approval.
- Do not imply that agreement status equals legal consent in production.
- Keep the UI copy clear: “candidate”, “manual verification”, “agreement state”.

## Exit criteria before AI

Only after this version is usable should the project move to `v0.1.5-ai-intake-gate`.
