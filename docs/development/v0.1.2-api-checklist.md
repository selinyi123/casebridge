# v0.1.2 API Checklist

> Scope: non-AI business skeleton. This file is the manual smoke-test checklist for the current key node.

## Expected API routes

- `GET /`
- `GET /api/v1/health`
- `GET /api/v1/clients`
- `GET /api/v1/clients/C-0001`
- `GET /api/v1/cases`
- `GET /api/v1/cases/CASE-0001`
- `GET /api/v1/cases/CASE-0001/notes`
- `POST /api/v1/cases/CASE-0001/notes`
- `GET /api/v1/resources`
- `POST /api/v1/resources/match`

## Manual curl checks

```bash
curl http://localhost:8000/api/v1/health
curl http://localhost:8000/api/v1/clients
curl http://localhost:8000/api/v1/clients/C-0001
curl http://localhost:8000/api/v1/cases/CASE-0001
curl http://localhost:8000/api/v1/resources
curl -X POST http://localhost:8000/api/v1/resources/match \
  -H 'Content-Type: application/json' \
  -d '{"need_tag_codes":["meal_difficulty","mobility_limited","living_alone"]}'
```

## Acceptance rules

- No AI provider is called.
- Resource candidates are selected by tag rules.
- Visit-note creation preserves `content_raw`.
- Redacted text is stored as `content_clean`.
- This version may use in-memory demo data; database migration is planned for v0.1.3.

## Known limitations

- No persistent database yet.
- No authentication yet.
- No formal CRUD forms in the web UI yet.
- PII redactor is a placeholder for synthetic demo use only.
