# v0.1.16.1 Reopen Workflow and Test Isolation Checklist

## Completed

- Reopen request model.
- Reopen repository.
- Reopen request migration.
- Reopen state router.
- Router mounted in FastAPI main app.
- Route-level reopen test using an isolated test case id.
- Reopen request create UI.

## API

- `GET /api/v1/cases/{case_id}/reopen-state/requests`
- `POST /api/v1/cases/{case_id}/reopen-state/requests`
- `POST /api/v1/cases/{case_id}/reopen-state/requests/{request_id}/review`
- `POST /api/v1/cases/{case_id}/reopen-state/reopen`

## Acceptance

- Reopen request creation does not reopen a case.
- Reopen request review does not reopen a case.
- Explicit reopen requires approved request.
- Explicit reopen requires `confirm_case_reopen=true`.
- Explicit reopen requires `responsibility_accepted=true`.
- Supervisor/admin role is required to review and reopen.
- AI does not reopen the case.

## Limitations

- Reopen review UI was blocked by safety checks.
- Route-level tests still use the shared app, but now use an isolated case id.
- Full test database isolation remains future work.
