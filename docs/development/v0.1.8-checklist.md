# v0.1.8 Redaction Gateway, Fixtures, and Apply Preview Checklist

> Scope: strengthen AI intake safety before any real provider or formal field apply.

## Expected behavior

- AI intake uses `run_redaction_gate()`.
- Redaction gate reports residual findings.
- Provider call is blocked if residual sensitive content remains.
- C-0001 golden intake fixture exists.
- Apply preview requires AI output review status `accepted` or `modified`.
- Apply preview does not write formal case fields.
- Apply preview writes an audit event.

## Expected API route

- `POST /api/v1/cases/CASE-0001/ai/outputs/{output_id}/apply-preview`

## Manual check

1. Open `/clients/c-0001/ai`.
2. Generate mock AI draft.
3. Try apply preview while output is `pending`.
4. Confirm preview is blocked.
5. Accept the draft.
6. Create apply preview.
7. Confirm preview says `will_write_formal_fields=false`.
8. Open `/clients/c-0001/timeline`.
9. Confirm `ai.apply_preview.created` audit event appears.

## Automated check

```bash
cd apps/api
pytest -q
```

Expected coverage:

- redaction gateway reports masked sensitive content
- golden fixture matches mock provider behavior
- pending AI output cannot create apply preview
- accepted AI output can create apply preview
- apply preview does not mutate formal case fields

## Known limitations

- Redaction gateway still wraps MVP redactor; it is not production DLP.
- Apply preview is not formal apply.
- No formal assessment target table yet.
- No RBAC/reviewer permissions yet.
- Provider remains mock only.
