# v0.1.9 Formal Assessment Target Checklist

> Scope: move from apply preview to explicit formal assessment target while preserving human responsibility and source trace.

## Expected API routes

- `GET /api/v1/cases/CASE-0001/assessments`
- `POST /api/v1/cases/CASE-0001/ai/outputs/{output_id}/apply-to-assessment`

## Manual flow

1. Open `/clients/c-0001/ai`.
2. Generate mock AI draft.
3. Accept the draft.
4. Create apply preview.
5. Check responsibility confirmation.
6. Apply to assessment.
7. Open `/clients/c-0001/assessments`.
8. Confirm formal assessment exists with source note, AI output, provider, prompt version, and reviewer.
9. Open `/clients/c-0001/timeline`.
10. Confirm `assessment` and `ai.output.applied_to_assessment` appear.

## Acceptance rules

- Pending AI output cannot be applied.
- Rejected AI output cannot be applied.
- Accepted or modified AI output can be applied only with responsibility confirmation.
- Formal assessment stores source trace.
- Original AI output remains unchanged.
- Apply action writes audit event.
- No service goal or resource link is auto-created.

## Known limitations

- Assessment schema is still generic JSON.
- No assessment edit/version history yet.
- No RBAC/reviewer role enforcement yet.
- No real external provider.
- No outcome tracking target yet.
