# v0.1.7 Provider and Prompt Gate Checklist

> Scope: centralize provider selection and prompt versioning before any real provider is enabled.

## Expected behavior

- AI intake uses `prompt_registry.py`.
- AI intake uses `provider_registry.py`.
- Only `mock` provider is enabled.
- Unknown providers raise `provider_not_enabled`.
- AI task stores provider and prompt version.
- Audit payload records provider and prompt version.

## Manual API check

Generate intake draft:

```bash
curl -X POST http://localhost:8000/api/v1/cases/CASE-0001/ai/intake \
  -H 'Content-Type: application/json' \
  -d '{"note_id":"NOTE-0001"}'
```

Expected response includes:

```json
{
  "provider": "mock",
  "prompt_version": "intake-v0.1.7"
}
```

## Automated checks

```bash
cd apps/api
pytest -q
```

Expected coverage:

- unknown provider is rejected
- AI intake response includes `provider=mock`
- AI intake response includes `prompt_version=intake-v0.1.7`
- timeline audit payload includes prompt version

## Acceptance rules

- No real external provider is called.
- Provider selection is not hardcoded in repository functions.
- Prompt version is not hidden inside free text.
- Provider and prompt version are visible in task/audit metadata.
- Formal case fields remain unchanged by AI draft generation or review.

## Known limitations

- No real provider configuration yet.
- No prompt test fixture set yet.
- Redaction is still MVP-level.
- AI output modification UI remains basic.
- Accepting a draft still does not apply it to formal assessment fields.
