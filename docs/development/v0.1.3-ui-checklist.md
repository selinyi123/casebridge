# v0.1.3 UI Checklist

> Scope: make the persistent manual C-0001 loop visible in the web app. No AI provider calls.

## Expected pages

- `/`
- `/clients/c-0001`

## Manual web checks

1. Start API on port 8000.
2. Start web app on port 3000.
3. Open `http://localhost:3000`.
4. Click `Open C-0001 workspace`.
5. Confirm client profile loads.
6. Confirm case summary loads.
7. Confirm timeline loads at least NOTE-0001.
8. Add a human note.
9. Confirm the note appears in timeline.
10. Click resource matching.
11. Confirm R-001 / R-003 / R-005 appear as candidate resources.

## Acceptance rules

- No AI provider is called.
- Resource matching remains deterministic.
- New notes are saved through API, not local-only UI state.
- Raw note text is displayed in the timeline.
- Candidate resources are labeled as requiring human verification.
- The UI must not imply automatic referral or automatic risk decision.

## Known limitations

- No service-goal UI yet.
- No referral creation UI yet.
- No authentication yet.
- No production design system yet.
- The page depends on API availability at `NEXT_PUBLIC_API_BASE_URL` or `http://localhost:8000`.
