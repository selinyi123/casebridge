# v0.1.16 Case Closure State Machine Checklist

## Completed

- Explicit closure transition repository function.
- Closure state router.
- Router mounted in FastAPI main app.
- Route-level test for explicit closure transition.

## API

- `POST /api/v1/cases/{case_id}/closure-state/close`

## Acceptance

- Closure draft must exist.
- Closure draft must be approved.
- Supervisor/admin role is required.
- `confirm_case_closure` must be true.
- `responsibility_accepted` must be true.
- Closing writes a `case.closed` audit event.
- AI does not close the case.

## Limitations

- Frontend explicit close page was blocked by safety checks.
- Route-level test mutates demo case status and should be isolated in future test database work.
- Reopen flow is not implemented yet.
