# v0.1.4 Manual Goal and Resource Link Checklist

> Scope: complete manual service-goal and resource-link workflow before AI integration.

## Expected API routes

- `GET /api/v1/cases/CASE-0001/goals`
- `POST /api/v1/cases/CASE-0001/goals`
- `GET /api/v1/cases/CASE-0001/referrals`
- `POST /api/v1/cases/CASE-0001/referrals`
- `PATCH /api/v1/cases/CASE-0001/referrals/{referral_id}/status`

## Manual API checks

Create a goal:

```bash
curl -X POST http://localhost:8000/api/v1/cases/CASE-0001/goals \
  -H 'Content-Type: application/json' \
  -d '{"title":"stable meal support","target_state":"verify meal resource and create follow-up arrangement"}'
```

Create a resource link candidate:

```bash
curl -X POST http://localhost:8000/api/v1/cases/CASE-0001/referrals \
  -H 'Content-Type: application/json' \
  -d '{"resource_code":"R-001","agreement_status":"none","notes":"candidate only"}'
```

## Agreement-state gate

Blocked transition when agreement state is missing:

```bash
curl -X PATCH http://localhost:8000/api/v1/cases/CASE-0001/referrals/REF-0001/status \
  -H 'Content-Type: application/json' \
  -d '{"status":"referred"}'
```

Expected: `409 agreement_required_for_referral_status`.

Allowed transition with explicit agreement state:

```bash
curl -X PATCH http://localhost:8000/api/v1/cases/CASE-0001/referrals/REF-0001/status \
  -H 'Content-Type: application/json' \
  -d '{"status":"referred","agreement_status":"verbal"}'
```

Expected: `200`.

## Manual web checks

1. Open `/clients/c-0001`.
2. Create a manual service goal.
3. Confirm the goal appears in the Service goals panel.
4. Run resource matching.
5. Create a manual resource link from a candidate.
6. Confirm the link appears in the Manual resource links panel.
7. Advance link with explicit agreement state.
8. Confirm link status changes to `referred`.

## Acceptance rules

- No AI provider is called.
- Matching result remains candidate-only.
- Manual resource link references an existing resource.
- High-commitment status transitions require explicit agreement state.
- Service goals are stored separately from notes.
- UI copy must not imply automatic approval or automatic service result.

## Known limitations

- Timeline still does not merge notes, goals, and resource links.
- Agreement state is not a production legal-consent model.
- No auth/RBAC yet.
- No AI intake gate yet.
